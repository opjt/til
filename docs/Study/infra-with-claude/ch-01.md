## GitOps에서 GitAIOps로

GitOps는 Git 저장소를 Single Source of Truth로 삼아, 인프라 상태를 선언적으로 관리하는 방법론을 의미한다.  

전통적인 배포 방식은 명령형으로 누군가 어떤 명령어로 어떻게 작업하였는지 기록하기 어려움이 있었지만  
GitOps는 Git 저장소(Ssot)를 통해 내가 원하는 자원의 상태를 선언하면 도구들이 그 상태를 맞추어 주는 방식이다.  

- 이게 가능한 이유는 쿠버네티스의 컨셉이 선언형 방식이기 때문이다  

GitAIOps는 이러한 GitOps에서 AI의 서포트를 받는 형식으로,  

트러블 슈팅 기록, 매니패스트 작성 어시스턴트, 문서화 등 과 같은 작업을 함께한다.  
오래된 시스템일 수록 문서화가 중요한데 AI-Agent 를 통하여 작업과 동시에 문서화와 검증을 수행함으로써 같이 일할 수 있는 환경을 만들어 준다.

작업 내역들을 Git 커밋으로 상세히 기록할 경우 따로 문서화하지 않아도 나중에 돌아보면 커밋들만으로 이력을 뒤져볼 수 있다

## 책의 구성과 실습 흐름

해당 책은 이론서가 아닌 실습 위주로 구성되어 있다.  

- 단순 이론만 적혀있는 것보단 실제 현업에서 생길만한 요구사항들을 적용해가면서 실습하는 것이 공부 흥미를 이끌어 준다.
  - 물론 부족한 이론 내용은 대충 넘기지 않고 채우는 것이 중요하다

GCP의 무료 크레딧을 통하여 GKE를 구성하고 챕터를 거듭하면서 가상의 `notiflex`라는 회사의 인프라를 성장시키는 흐름이다.

## 실습 가드레일

책에서는 독자가 입력해야 할 명령어를 알려주지 않고 에이전트에게 전달할 자연어를 알려준다.

만약 사전에 준비된 가드레일이 없었더라면 독자마다 다른 방향으로 실습을 하게 될 것이다.  

<https://github.com/sysnet4admin/_Book_GitAIOps> 책 실습 레포의 CLAUDE.md를 살펴보면 독자가 입력한 자연어와 최대한 관련된 항목의 참조파일을 찾아 수행한다.

- `prompt-guardrails`: 실행 지침, 사전 조건, 진행 절차들의 명시됨
- `decision-guides`: 도구의 선택 근거
- `result-templates`: 체크리스트 검증, 실행 결과 확인

이러한 사전 가드레일 덕분에 각 챕터마다 빠진 부분 없이 시나리오를 수행하고 제대로 가고 있는지 검증할 수 있다.
